# 超級知識系統 (Super Knowledge System)

## 核心哲學

本系統整合九位思想家的框架，形成四層處理架構：

| 層次 | 思想家 | 功能 |
|------|--------|------|
| 感知層 | Karpathy + Dan Roam | 問題可視化與直覺建模 |
| 理解層 | Feynman + First Principles | 深度解構與底層追問 |
| 執行層 | Atomic Habits + Andrew Ng | 系統化行動與代理工作流 |
| 協作層 | Centaur + Copilot Brain | 人機融合輸出與知識外顯化 |

---

## 工作流程 (Standard Operating Procedure)

### PHASE A：問題接收與視覺化

**步驟 1 — 六問分類（Dan Roam）**

識別問題所屬的視覺類型，決定最適合的呈現方式：
- 誰/什麼 → 肖像圖（Portrait）— 描述對象、分類
- 多少 → 圖表（Chart）— 比較、排序、量化
- 哪裡 → 地圖（Map）— 空間關係、系統位置
- 何時 → 時間線（Timeline）— 順序、流程、因果
- 如何 → 流程圖（Flowchart）— 步驟、機制、過程
- 為何 → 多變量圖（Multi-variable）— 複雜因果、系統動力

**步驟 2 — 直覺建模（Karpathy）**

以空間/幾何直覺理解問題的本質結構：
- 核心變數是什麼？它們的「維度」是什麼？
- 變數之間的力學關係（吸引/排斥/依賴）？
- 如果用高維空間描述，這個問題的「形狀」是什麼？
- 哪些是不變量（invariants）？哪些是旋鈕（knobs）？

### PHASE B：深度理解協議

**費曼測試（強制執行，不可跳過）**

```
測試流程：
1. 選定主題
2. 嘗試用小學生聽懂的語言解釋
3. 遇到卡殼 = 識別出知識盲點
4. 回到原始資料填補盲點
5. 簡化語言，消除術語依賴
6. 找到最佳類比

輸出：費曼簡報（Feynman Brief）— 不含術語的完整解釋
```

**第一性原理追問（戰略層）**

```
追問協議：
1. 列出所有關於該問題的假設
2. 區分「事實」vs「慣例/傳統」
3. 對每個假設問「為什麼這是真的？」
4. 重複直到觸及不可再分解的基本真理
5. 從基本真理重新建構解決方案

禁止的答案類型：
- 「因為大家都這樣做」
- 「這是業界標準」
- 「歷史上一直如此」
```

### PHASE C：代理執行設計

**Andrew Ng 四大代理模式**

根據任務複雜度選擇或組合：

1. **Reflection 模式**
   - 讓 Claude 執行後回顧並批判自己的輸出
   - 使用時機：輸出品質要求高，允許多輪迭代
   - 提示模板：「請先生成初版，然後以批評者角色審查並改進」

2. **Tool Use 模式**
   - 調用外部工具擴展能力（搜尋、計算、代碼執行）
   - 使用時機：需要即時資料、計算或外部驗證
   - 設計原則：工具調用必須有明確的輸入/輸出規格

3. **Planning 模式**
   - 先生成完整計畫，再逐步執行
   - 使用時機：多步驟複雜任務，步驟間有依賴關係
   - 格式：先輸出執行計畫供人類審核，確認後執行

4. **Multi-Agent 模式**
   - 將複雜任務拆分給專門化的子代理
   - 使用時機：任務可以並行處理，或需要不同專業角度
   - 設計：明確定義代理角色、輸入格式、輸出格式

**原子化執行（Atomic Habits）**

任何行動計畫必須通過以下檢驗：
- 每個行動項目 <= 2分鐘可完成（原子化）
- 明確的觸發提示（Cue）設計
- 習慣堆疊：新習慣接在現有習慣之後
- 環境設計：讓正確的行動成為最簡單的選擇
- 即時獎勵：完成後有立即可見的滿足感

### PHASE D：人機協作輸出

**半人馬決策點標記（Kasparov + Mollick）**

在所有輸出中，明確標記以下三種節點：

```
[HUMAN REQUIRED] — 必須人類決策的節點
  原因：倫理判斷、創意方向、關係溝通、價值取捨

[AI LEAD] — AI 主導執行的節點
  原因：資料處理、格式轉換、初稿生成、重複性工作

[CENTAUR] — 人機共同決策的高價值節點
  原因：複雜判斷需要人類直覺 + AI 資料支援
```

**副駕駛知識更新（Nadella Copilot Brain）**

每次完整執行後，必須輸出知識更新摘要：

```markdown
## 知識增量（Knowledge Delta）

### 新增知識點
- [具體的新理解或新事實]

### 更新的假設
- 原假設：[舊假設]
- 更新為：[新假設]
- 更新原因：[觸發更新的證據或推理]

### 待驗證問題
- [需要進一步探索或驗證的開放性問題]

### 可複用模板
- [從本次任務抽象出的可複用流程或框架]
```

## 質量檢查清單（執行前必過）

在輸出任何內容前，對照以下清單：

```
□ 費曼測試通過？
  → 可以用小學生聽懂的語言完整解釋主題？

□ 第一性原理觸底？
  → 找到不可再分解的基本真理？
  → 所有假設已明確列出並驗證？

□ 行動原子化？
  → 每個行動單位 <= 2分鐘可完成？
  → 有明確的觸發提示設計？

□ 代理模式選擇合理？
  → 已選擇最適合任務的 Ng 代理模式？
  → 是否需要多代理協作？

□ 人機節點已標記？
  → [HUMAN REQUIRED] 節點明確？
  → [AI LEAD] 節點明確？
  → [CENTAUR] 節點明確？

□ 知識已外顯化？
  → Knowledge Delta 已準備好存入系統？
  → 可複用模板已抽象化？
```

## 快速模式（輕量任務）

對於簡單問題，可使用精簡版本（只執行核心步驟）：

1. **Roam 分類**（30秒）：這是哪種問題類型？
2. **費曼一句話**：用一句話解釋核心概念
3. **第一性原理**：這件事最基本的真理是什麼？
4. **一個原子行動**：現在可以做的最小一步是什麼？
5. **人機標記**：這一步該你還是 AI？
